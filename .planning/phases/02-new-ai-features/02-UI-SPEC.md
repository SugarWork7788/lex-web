# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2 (Intel search v2 result cards + Audit PDF download). Generated by gsd-ui-researcher; verified by gsd-ui-checker.
>
> **Surfaces in scope:** `/intel/search` (top-5 best-matches section + per-source breakdown), `/audit` (single "Download as PDF" affordance + post-click feedback). All other surfaces remain unchanged.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (hand-rolled in `app/components/*` and per-route `app/<route>/*.tsx`) |
| Preset | not applicable (no shadcn — `components.json` confirmed absent 2026-05-10) |
| Component library | none (raw HTML + Tailwind utilities; semantic elements: `<details>`, `<form>`, `<a>`, `<button>`) |
| Icon library | Inline Unicode glyphs (`✦ → ↗ ▼ 🔗 ⏱ ↻ ←`); no `lucide` / `heroicons` dependency. Phase 2 keeps this convention. |
| Font | `--font-serif` (Georgia/Source Serif) for headings; `--font-sans` for body/UI; both declared in `app/globals.css @theme` and bound to `font-serif` / default |
| CSS framework | Tailwind 4 (CSS-config via `@import "tailwindcss"` + `@theme inline`; no `tailwind.config.ts`) |
| Color scheme | Dark-first (`bg-stone-950 text-stone-100`); print path neutralises stone-* to white per `globals.css @media print` |
| Locale | Bulgarian Cyrillic (`bg-BG`); all visible UI strings are Bulgarian; `toLocaleString("bg-BG")` for numbers |

**Reuse mandate:** No new layout primitive may be introduced unless it cannot be expressed as a stack of utility classes already used on `/audit/page.tsx` or `/intel/search/page.tsx`. The two new component files this phase introduces are documented under "Component Inventory" below.

### Voice rules (matched against existing UI)

- Imperative formal: "Опитайте" not "Опитай" in body copy; "Опитай отново" allowed on a button (one-verb labels stay terse — matches "Затвори" / "Търси" precedent).
- No exclamation marks anywhere. The existing UI uses periods or none.
- Numbers: `toLocaleString("bg-BG")` for any count or amount; never bare digits with commas.
- Time: "след Ns" pattern for countdowns (matches `RateLimitToast`); "~10 секунди" with tilde for estimates.
- Labels stay short: ≤3 Bulgarian words for buttons, ≤6 words for headings, ≤14 words for helper text.

### Destructive actions

None in Phase 2. The PDF download is a read-only action; there is no DELETE / cancel-subscription / data-loss flow. No confirmation dialog needed.

```tsx
  <div className="flex shrink-0 items-center gap-3">
    <button type="button" onClick={onRetry}
```

Differences from `RateLimitToast`:
- `role="alert"` + `aria-live="assertive"` (rate-limit is "polite"; PDF failure is more urgent — user is mid-task).
- Red palette instead of amber (this is an error, not a throttle).
- Two action buttons (retry + dismiss) instead of one (dismiss).
- Multiline body — heading + explanation.

### Loading micro-UX during AI quote extraction (D-03 / D-04)

**Decision: per-card text placeholder (NOT skeleton blocks, NOT sentence-by-sentence stream).**

Rationale:
- Articles are at most 1–2 sentences long; sentence-by-sentence streaming animation has no perceivable benefit at that length.
- A skeleton-block approach would require introducing animated rectangles — a new primitive that doesn't exist in the project. Adds complexity for ~1s of UX.
- The existing summary card (`intel-search-summary.tsx:134`) uses the exact same pattern — `italic animate-pulse` text placeholder during streaming. Phase 2 inherits that precedent.

Concretely, while the Haiku call is in flight the quote slot shows:

```tsx
<p className="text-sm text-stone-400 italic animate-pulse">Извличам цитати…</p>
```

When the response streams back, the placeholder is replaced by the quote text token-by-token (same `TextDecoder` reader pattern used in `intel-search-summary.tsx:103–112`). A streaming cursor (`bg-red-500` pulse) renders at the tail until the stream closes.

**Source-row-verbatim cards** (sanctions, offshore, OLAF, prosecution, NAP — D-03) have NO loading state. They render at first paint with the data the server already has. Only article cards stream.

### `/audit` layout — placement of "Свали като PDF" button

**Decision: in the page header, right-aligned below the description paragraph, on the same baseline as the first `<Stat>` row.**

Visual location:
- The existing header is a left-aligned block inside `border-b border-stone-800 pb-6` (`app/audit/page.tsx:78–89`).
- The first row of `<Stat>` items follows immediately at `mt-6 flex flex-wrap gap-x-4 gap-y-2 text-sm` (line 91).
- The button slots in as a sibling to the `<ul>` of stats, on the right edge: `mt-6 flex flex-wrap items-center justify-between gap-4`. The stats wrap on narrow viewports; the button stays on its own line below them on `<sm` breakpoints.

Why here (not in the page H1 row, not floating, not next to the timeline):
- It's an export action — semantically tied to "what's on this page", which is the stats summary.
- Placing it next to the H1 would compete with the H1's typographic weight (`font-serif text-3xl`). The button would read as a co-headline.
- Placing it next to TimelineRoadmap would imply it exports only the timeline; the action is whole-page.
- The right edge of the stats row keeps it discoverable on first view (above the fold on desktop) and out of the way on mobile (drops below the stats).

Concrete markup outline:

```tsx
<div className="mt-6 flex flex-wrap items-center justify-between gap-4 print:hidden">
  <ul className="flex flex-wrap gap-x-4 gap-y-2 text-sm">
```

## Accessibility Contract

| Requirement | Phase 2 commitment |
|-------------|-------------------|
| Keyboard navigation | Best-match cards: title link is tabbable; "Виж в раздела →" link is tabbable; external-link `↗` is tabbable. Tab order: title → quote (skipped — no focusable child) → meta links → external link. Card itself is not a tab stop. PDF button: native `<button>`, single tab stop. |
| Focus visibility | All interactive elements use `focus-visible:outline outline-2 outline-offset-2 outline-red-500` (matches existing audit/intel buttons). Cards use `focus-within:ring-1 ring-red-500/30` to surface child-focus on the card boundary. |
| Screen reader | Section header is a real `<h2>`. Source pill is `<span>` with sr-text — pill text is the `aria-label` of the parent card link if title is short, otherwise pill stays decorative. AI-quote streaming uses `aria-live="polite"` on the quote `<p>` so SR users hear the quote when it finishes (debounced — only the final text is announced, not every token). PDF button's loading state announces via `aria-live="polite"` once on idle→loading and once on loading→done. |
| Color independence | Source-pill colors are NEVER the only signal — every pill has Bulgarian text. Severity dots on `/audit` (existing) already have text labels. The streaming-cursor pulse is decorative (`aria-hidden`). |
| Motion | Hover transitions use `transition-colors` (200ms). Streaming cursor uses `animate-pulse` (1s loop, Tailwind default). No flash, no parallax, no autoplay. Respects `prefers-reduced-motion` via Tailwind 4's built-in `motion-reduce:` variant — Phase 2 does NOT introduce any animation that survives `motion-reduce`. |
| Touch targets | Button height ≥ 40px (`py-2 text-sm` = ~40px); helper-text below has its own line so tap area isn't crowded. Card title links span a generous height (multi-line title + leading-snug ≈ 44–60px). |
| `aria-live` regions | Best-matches AI quote: `aria-live="polite"` on the quote `<p>` (set via attribute on the streaming target, not on a wrapper, to keep rate-limit toast as the only `role="status"` region). PDF button: an `sr-only` `<span aria-live="polite">` inside the button announces state transitions. PDF error toast: `role="alert"` + `aria-live="assertive"` (urgency outranks the rate-limit toast's "polite"). |
| Reduced-motion fallback | If the user has `prefers-reduced-motion: reduce`, the streaming cursor stops pulsing and stays visible until stream end (handled by Tailwind's automatic `motion-reduce` cascade — `animate-pulse` becomes a no-op). The "Извличам цитати…" placeholder loses its `animate-pulse` automatically. |

## Empty / Edge States

| State | UI |
|-------|----|
| Search has 0 cross-source hits AND 0 per-source hits | Existing per-source breakdown shows "Няма попадения." for each section (existing copy at `app/intel/search/page.tsx:179`); best-matches section is HIDDEN. AI summary card still streams its observations. No new empty-state copy. |
| Search has 0 cross-source ranked hits BUT some per-source hits | Best-matches HIDDEN (D-01). Per-source breakdown renders normally. Page reads as "no clear top matches; here are sliced views by source." |
| Search has 1–4 cross-source ranked hits | Best-matches renders with whatever ranks (D-01). No "fewer results than expected" copy — silent. |
| Search has 5 cross-source ranked hits | Best-matches caps at 5 (D-01). |
| AI quote extraction fails for one card (article only) | Card renders with title + meta + fallback `<p className="text-xs text-stone-500 italic">Цитатът не може да бъде извлечен. Виж пълния текст в раздела.</p>` in the quote slot. Other cards unaffected. |
| AI quote extraction is rate-limited (429) | The existing `RateLimitToast` shows. The triggering card displays the same per-card fallback copy. |
| User searches with empty query | Existing form-required attribute prevents submission (no change). |
| User searches with 1-character query | (Existing behavior — server returns whatever ILIKE/tsvector matches; UI does not validate.) |
| Audit PDF: server returns 5xx | Error toast appears; button reverts to idle; user can retry. |
| Audit PDF: server takes >10s (success criterion breach) | Loading state continues until the response settles. If `maxDuration: 60` is hit, the response is a 504/500 — error toast. No client-side timeout shorter than the server's `maxDuration`. |
| Audit PDF: 429 rate-limit | Existing `RateLimitToast` shows (same hook). Button stays in idle state (the throttle implies the click never reached the renderer). |
| Audit PDF: blob received but browser blocks download | Browser-level prompt is the OS layer; not our concern. The fetch resolved successfully so we show "Свален ✓" — if the user's browser denied, they see a permission dialog and can re-trigger. |
