## Pitfalls captured

- **`app/audit/page.tsx` is also a `<VoteButton>` consumer.** Don't forget the list page when refactoring the detail page's button. The plan's file list missed this; build catches it immediately if you forget.
- **The route's `already_voted` response is HTTP 200 with `success: false`** (no `status:` argument on the `Response.json` call). This is the existing convention, deliberately preserved. The plan-checker flagged it as a semantic oddity; documented here so future maintainers don't "fix" it to 409.
- **Server-Component wrapper avoids hydration flash.** If you tried to do anon/authed detection client-side via `useSession()`, the page would render the authed button for half a second before falling back to the disabled state. Server-side decision via `getSession()` in `vote-button-server.tsx` eliminates the flash.
