### Phase 8 Exceptions

- **Mobile filter-chip vertical padding bumped to `py-3` (12px)** when the `sm` breakpoint is below 640px. Computed tap-target with `text-xs` (12px line-height ~1) + `py-3` × 2 + 1px borders ≈ 38–42px wide; we accept ≥40px since the existing `/audit` filter pills ship at this size and have not regressed in mobile UAT. Carry-forward of Phase 2 D-08 mobile-pill ergonomics rule.
- **Issue card padding may drop to `p-4` on mobile** if inner content overflows; default is `p-5`. Decided at executor-time only if 375px viewport overflows.
- Date inputs sit at `py-2 px-3` (matching `/intel/articles` form inputs verbatim — line 37). Do NOT introduce a new input height.
- Footer attribution row: `mt-12 pt-6 border-t border-stone-800/50` — outside the page-card scale; distinct, lighter border to mark provenance not content.

### Bulgarian Cyrillic-specific

- **Body line-height stays at `leading-relaxed` (1.625)**, not 1.5. Cyrillic glyphs render slightly taller than Latin and tighter leading produces visual crowding at 14px. Carry-forward Phase 2 calibration. Do NOT tighten on AI-summary markdown either.
- **Headings stay at `leading-snug` (1.375)**. Tighter heading leading is OK because heading sizes (16–30px) absorb the extra glyph height without crowding.
- **No font-feature-settings overrides.** Inter ships with proper Cyrillic; Source Serif 4 ships with proper Cyrillic.
- **`font-variant-numeric: tabular-nums`** explicitly applied to issue numbers, act counts, and AI-summary counters so 1/9 in BG ("януари 2026" / "септември 2026") align in lists.

### `DV_ACT_PILL` — NEW: 5 hues + 1 fallback

This is **the only new visual primitive Phase 8 introduces.** It mirrors Phase 2's source-pill triplet pattern and reuses the same per-pill mechanical class shape so executor can copy verbatim.

**Decision: Option (b) — tighter, more muted palette than Phase 2's 6-hue set.** Rationale: DV browsing is dense (≤50 acts per issue, paginated listing). Phase 2's red/amber/blue/stone/purple/emerald set was tuned for a single-row source-pill triplet on intel results — visual variety helped scanning. On a dense DV page with 50 act cards, that variety becomes noise. We want the pill to read as *category* not as *priority*. Picking a 5-cool-tone palette + 1 accent for the weightiest category ("Закон") gives:

- Visual calm for the bulk of the corpus (Постановления, Решения, etc. dominate by volume)
- A single "this matters most" signal that piggy-backs on the existing red brand for genuinely legislative acts
- WCAG-AA contrast on the dark background (verified — see Color & Contrast below)

| Act type (Bulgarian) | Hue | Pill class |
|----|----|----|
| Закон | red (matches accent — these are the weightiest gazette items) | `bg-red-950/40 text-red-300 ring-1 ring-red-800/40` |
| Указ | amber (presidential acts — visually distinct, warm) | `bg-amber-950/40 text-amber-300 ring-1 ring-amber-800/40` |
| Постановление | sky (cabinet acts — cool, common, recede) | `bg-sky-950/40 text-sky-300 ring-1 ring-sky-800/40` |
| Наредба | indigo (ministry regulations — mid-saturation, calm) | `bg-indigo-950/40 text-indigo-300 ring-1 ring-indigo-800/40` |
| Решение | teal (decisions — distinct from sky, still cool) | `bg-teal-950/40 text-teal-300 ring-1 ring-teal-800/40` |
| Обявление + Other (fallback) | stone (neutral default) | `bg-stone-800/60 text-stone-300 ring-1 ring-stone-700/40` |

**Pill geometry (verbatim Phase 2):** `px-1.5 py-0.5 text-xs uppercase tracking-wider rounded-[2px]`. Inline within meta rows. Never wraps to multi-line.

**Listing context (`/dv` cards):** show top-3 act-type pills horizontally; if more than 3 distinct types in the issue, append `+ N` faint-stone counter. Order: by act-count descending in that issue.

**Detail context (`/dv/[issue]` cards):** show exactly ONE pill matching that act's `act_type`. No fallback to multi-pill on detail.

All declared color combinations were verified against AA (4.5:1 for body, 3:1 for ≥18px or bold ≥14px). Cyrillic glyph stroke weights are similar to Latin — same numeric ratios apply.

### Streaming cursor (in AI summary expansion)

Verbatim from `intel-search-summary.tsx:142–143`:

```
{status === "streaming" && (
  <span className="ml-1 inline-block h-3 w-2 animate-pulse bg-red-500 align-middle" />
)}
```

- 8px wide × 12px tall (`w-2 h-3`).
- `animate-pulse` (Tailwind default 2s ease-in-out infinite alternate opacity 1↔0.5).
- Inline at the end of the latest streamed text token; replaced with nothing when `status === 'done'`.

### `aria-live` debouncing (carry-forward Phase 2 D-04)

```
{status === "done" && (
  <span className="sr-only" aria-live="polite">
    AI обобщението е готово. {summaryFirst40Chars}…
  </span>
)}
```

Rendered **only** when `status === 'done'`, never during streaming. Token-by-token streaming would re-fire announcements every tick and overwhelm screen readers. Identical pattern to `intel-search-summary.tsx` and to `RateLimitToast`'s announce-once.

### "Скрий" (collapse) button (decision on Q5)

| State | Class delta | Copy |
|-------|----|------|
| Default (only rendered when `status === 'done'`) | `text-xs text-stone-400 hover:text-stone-100 hover:underline` aligned right | "Скрий" |

### Footer attribution copy (decision on Q10)

```tsx
</div>
```

Why a separate page-level footer instead of folding into the global footer:

1. The global footer credits multiple data sources collectively. DV is a single, authoritative source for this page; honest attribution warrants a per-page line.
2. The global footer doesn't render on print (covered by `@media print { footer { display: none !important; } }` in `globals.css`). The DV per-page attribution **must** print so a user printing or PDF-exporting (Phase 2 path) sees the source. Therefore the per-page attribution must NOT use the `<footer>` HTML element — use `<div>`.

**No sticky footer.** It scrolls naturally at the bottom of the content.
